## 2. 任务 D1-3：调研 Zpix 字体 + 字符子集化

### 2.1 目标

调研清楚 **Zpix**（或其他 CC0 中文字素字体）从哪下、怎么字符子集化（减小体积）、怎么集成。**不下载、不集成**——只出调研报告，让 orchestrator 决定要不要采纳 + 派 Tab 1 实施 D1-6。

### 2.2 必调研项

1. **字体源**：
   - GitHub（搜 `zpix` / `fusion pixel font` / `bitmap chinese`）
   - OpenGameArt / itch.io CC0 区
   - 候选 1-2 个（最好对比表）
2. **许可证**：CC0 / CC-BY / Public Domain（**不接受 GPL / 商业版权**）
3. **字符子集化方案**：
   - 目标：GB2312 一级 3755 字 + 标点 = ~4000 字符
   - 工具：`fonttools` / `pyftsubset` / `subset-font`（npm）任选
   - 步骤：命令行命令
4. **集成方案**：
   - 放在 `assets/font/zpix.woff2`
   - `@font-face` 在 `src/renderer/index.html`
   - CSS `font-family: 'zpix', monospace`
   - 替换现有 Press Start 2P 的引用
5. **风险**：
   - 字符数 / 版权
   - 子集化后丢失字符的回退方案
   - 集成坑（line-height / 字符宽 / 中文标点位置）
6. **耗时估计**：下载 + 子集化 + 集成 预计多少分钟
