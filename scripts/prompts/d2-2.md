## 2. 任务 D2-2：音频系统 (BGM fade + SFX)

### 2.1 背景

`docs/dev-plan-full.md` §3.5 + §2.6 已定义：

- **BGM**：8-12 首，90s 复古合成器 + 钢琴 + 日本民谣采样，AI 生成（Suno v3.5+）
- **SFX**：20-30 个，门 / 风铃 / 列车鸣笛 / 胶片倒带 / 妖怪低鸣，AI 生成（Stable Audio）+ freesound CC0 兜底
- **UI 音**：5-8 个，翻页 / 选项 / 存档 / 错误
- **本 sprint 范围**：3 BGM + 11 SFX（prompt 已在 `docs/research/ai-audio-prompts.md`，等用户手动生成后落地）
- 关键设计原则（CLAUDE.md）：**突发音效比循环 BGM 更重要**——氛围靠"对的事件 + 对的音"

### 2.2 必做步骤（估时 2h）

1. **建目录结构**（~10 min）
   ```
   src/audio/
     index.ts          # 统一导出 (bgm, sfx, master)
     bgm.ts            # BGM 播放器 + crossfade
     sfx.ts            # SFX one-shot
     master.ts         # 主音量 + 静音
     loader.ts         # 资产加载 + 缓存
     types.ts          # AudioTrack, PlayOptions
   src/preload/
     audio-api.ts      # 暴露给渲染进程
   src/renderer/src/
     audio-init.ts     # 启动时按 scene 触发 BGM
   src/audio/__tests__/
     bgm.test.ts
     sfx.test.ts
   scripts/
     postprocess-audio.sh   # 现有 (D1-4.5 写好), 检查可用
   ```

2. **类型定义**（~5 min）
   ```ts
   export interface AudioTrack {
     id: string;           // 'bgm/prologue_anomaly'
     path: string;         // 'assets/audio/bgm/prologue_anomaly.ogg'
     durationMs: number;
     loop: boolean;        // BGM = true, SFX = false
   }
   export interface PlayOptions {
     volume?: number;      // 0.0-1.0
     loop?: boolean;
     fadeInMs?: number;    // BGM 默认 2000
     fadeOutMs?: number;   // BGM 切换时
   }
   ```

3. **loader**（~15 min）
   - 启动时扫 `assets/audio/bgm/*.ogg` + `assets/audio/sfx/*.ogg` 建索引
   - 输出 `assets/audio/index.json` 供运行时读（**不要**运行时扫盘，Electron 启动慢）
   - 提供 `getTrack(id): AudioTrack`

4. **bgm.ts**（~25 min）
   - 关键 API：`bgm.crossfade(track: string, durationMs = 2000): Promise<void>`
   - 实现：
     - 当前 BGM 音量从 master.linearRampToValueAtTime(vol, ctx.currentTime + fadeOut/1000)
     - 加载新 BGM → play → volume 从 0 linearRampToValueAtTime(vol, ctx.currentTime + fadeIn/1000)
     - 老 BGM 淡完 stop
   - **不**支持同时多 BGM（一个 AudioContext 一个 bgmGain）
   - vitest：mock AudioContext，验证 crossfade 时序

5. **sfx.ts**（~20 min）
   - API：`sfx.play(track: string, opts?: PlayOptions): void`
   - 每次新建一个 `AudioBufferSourceNode`（**不复用**，用完即弃）
   - SFX 通道独立（`sfxGain`），不打断 BGM
   - 限制：同时最多 8 个 SFX 节点（超过丢弃最老的，防爆音）
   - vitest：mock AudioContext，验证 8+ play 不爆

6. **master.ts**（~10 min）
   - `audio.setMasterVolume(0.0-1.0)`：linearRamp 到目标
   - `audio.mute()` / `audio.unmute()`：toggle
   - 持久化到 `localStorage`（用户设置音量下次还在）

7. **preload 暴露 API**（~10 min）
   - `window.audio = { bgm.crossfade, sfx.play, master.setVolume, master.mute }`
   - **不要**把整个 AudioContext 暴露到 renderer（安全 + 性能）

8. **scene 集成**（~15 min）
   - `src/renderer/src/states/CHAPTER_X.ts` 进入新 scene 时：
     - 如果 `scene.bgm` 不同 → `bgm.crossfade(scene.bgm, 2000)`
     - `scene.sfx_on_enter` 数组 → 逐个 `sfx.play()`
   - yaml 字段（`scripts/yaml/chapter-1.yaml` 已定义）：`bgm: <id>`, `sfx_on_enter: [<id>...]`

9. **资产落地**（占位, ~10 min）
   - **本任务不生成音频**（用户手动跑 Suno + Stable Audio + ffmpeg）
   - 但代码要**能跑**——加 `assets/audio/.gitkeep` + `assets/audio/index.json`（空索引）
   - 写 `docs/research/ai-audio-prompts.md` 资产追踪表的"待生成"提示
   - 写脚本：`scripts/check-audio-assets.sh` 列出 `ai-audio-prompts.md` 标 `⏳` 但 `assets/audio/` 还没文件的 id（手动跑，对照填）

10. **跑门禁**
    ```bash

    pnpm exec biome format --write .
    pnpm build
    ```

11. **Commit（1 个）**
    - 标题：`feat: 音频系统 (BGM crossfade + SFX + master) (D2-2)`
    - body 列改的文件 + API 清单 + 资产占位说明

## 3. 已知陷阱（CLAUDE.md 22+ 条必读）

完整列表见工作目录的 `CLAUDE.md`。**重点关注**：

- **pnpm only**
- **Electron AudioContext**：必须在 `window` 用户首次交互后 resume（autoplay policy），**renderer 启动时不能自动 play**
- **相对路径 off-by-one**：`src/audio/...`（在 `src/` 下）到 `assets/audio/` 是 `../assets/audio/...`（**1 层**）
- **vite 静态资源**：`assets/audio/*.ogg` 必须在 `electron.vite.config.ts` 的 `assets` 包含，否则 build 不带
- biome format 用 `--write`（见 [[ci-autocrlf-noop]]）
- vitest happy-dom 15 不实现 AudioContext → mock
- **electron-builder 24 Windows symlink**：开发模式可以，生产 pack 要 Developer Mode
- PowerShell heredoc 转义
- `.git/` 路径 safety hook

**D2-2 特有**：

- **本任务不生成音频文件**（你只写代码）。AI 生成是 orchestrator + 用户手动跑，**别**尝试在 CI 或本地脚本里调 Suno/Stable Audio API。
- **ffmpeg 转 OGG** 在 `scripts/postprocess-audio.sh` 已有（D1-4.5），**别**重新写。
